# Vulkan渲染器架构设计方案

## 背景与目标

用户有图形学经验（实现过CUDA Path Tracer），刚学会Vulkan，希望构建一个渲染器用于复现各种渲染技术，最终目标是实现HSGI（Hierarchical Sparse Global Illumination）。

## 核心设计原则

基于技术复现的需求，架构应遵循：

1. **快速迭代** - Shader热重载、参数实时调整
2. **技术隔离** - 不同渲染技术可独立实现、切换
3. **可视化调试** - GPU Profiler、Texture Inspector、ImGui集成
4. **渐进式复杂度** - 从基础开始，支持高级特性扩展

## 推荐架构层次

```
┌─────────────────────────────────────────────────────────────┐
│                    Application Layer                        │

├─────────────────────────────────────────────────────────────┤

│    (Instance/Device/Swapchain/Queue/CommandBuffer封装)       │
└─────────────────────────────────────────────────────────────┘
```

## 关键封装建议

### 1. Vulkan Core Layer - 最小化封装

**不要过度封装Vulkan对象**，保持Vk*句柄可访问，只封装生命周期管理：

```cpp
// 好的封装：RAII + 直接访问底层句柄
class Buffer {
public:
    VkBuffer handle() const { return m_buffer; }
    VkDeviceMemory memory() const { return m_memory; }
    void* mapped() const { return m_mapped; }
    
    // 便捷方法，但底层都暴露
    void upload(const void* data, size_t size, size_t offset = 0);
    void flush();
    
private:
    VkBuffer m_buffer;
    VkDeviceMemory m_memory;
    void* m_mapped = nullptr;
};
```

**核心类：**
- `Context`: Instance + PhysicalDevice + Device + Queues
- `Swapchain`: 窗口呈现，支持动态调整大小
- `CommandPool`/`CommandBuffer`: 命令录制
- `Buffer`/`Image`: GPU资源，支持VMA（VulkanMemoryAllocator）

### 2. Resource Layer - 描述符管理

**使用DescriptorSetLayout缓存**，避免重复创建：

```cpp
// 基于哈希的Layout缓存
DescriptorSetLayoutHandle createDescriptorSetLayout(
    const std::vector<VkDescriptorSetLayoutBinding>& bindings);

// 使用DSL（Domain Specific Language）简化描述符定义
DescriptorSetBuilder dsBuilder;
dsBuilder.bindUniformBuffer(0, cameraBuffer)
         .bindCombinedImageSampler(1, albedoTexture)
         .build(pipelineLayout);
```

### 3. RenderGraph - 最重要的封装

**RenderGraph是现代渲染器的核心**，强烈推荐实现：

```cpp
// 声明式定义渲染流程
renderGraph.addPass("GBuffer", [&](RenderPassBuilder& builder) {
    builder.writeColor(0, "Albedo", Format::RGBA8)
           .writeColor(1, "Normal", Format::RGBA16F)
           .writeDepth("Depth")
           .execute([](CommandBuffer& cmd, const RenderPassResources& res) {
               // 录制GBuffer绘制命令
           });
});

renderGraph.addPass("DeferredLighting", [&](RenderPassBuilder& builder) {
    builder.read("Albedo")
           .read("Normal")
           .read("Depth")
           .writeColor(0, "LitScene")
           .execute([](CommandBuffer& cmd, const RenderPassResources& res) {
               // 全屏光照计算
           });
});

renderGraph.compile(); // 自动处理资源分配、屏障、内存别名
renderGraph.execute(cmd);
```

**RenderGraph优势：**
- 自动处理Image Layout转换和内存屏障
- 支持 transient resource 内存复用
- 便于实现不同技术的A/B对比
- 可视化整个渲染流程

### 4. Technique Layer - 技术插件化

每个渲染技术实现统一接口：

```cpp
class IRenderTechnique {
public:
    virtual ~IRenderTechnique() = default;
    virtual void init(RenderContext& ctx, RenderGraph& graph) = 0;
    virtual void render(RenderGraph& graph, const Scene& scene, const Camera& camera) = 0;
    virtual void onResize(uint32_t width, uint32_t height) = 0;
    virtual void drawUI() = 0; // ImGui调试面板
    virtual const char* name() const = 0;
};

// 具体实现
class DeferredPBR : public IRenderTechnique { ... };
class HSGI : public IRenderTechnique { ... };
```

### 5. Shader管理 - 热重载

```cpp
class ShaderLibrary {
public:
    // 自动监听文件变化
    void watchAndReload();
    
    // 编译缓存
    VkShaderModule load(const std::filesystem::path& path, 
                        ShaderStage stage,
                        const std::vector<std::string>& defines = {});
    
    // 获取SPIRV反射信息
    const ShaderReflection& getReflection(const std::string& name) const;
};
```

使用`shaderc`或调用`glslangValidator`运行时编译，或集成`slang`支持更多特性。

## 推荐第三方库

| 功能 | 推荐库 | 理由 |
|------|--------|------|
| 窗口/输入 | `glfw` | 标准选择，支持多平台 |

| 数学库 | `glm` 或 `DirectXMath` | glm更熟悉，DXMath SIMD优化更好 |

| SPIRV反射 | `SPIRV-Reflect` | 自动生成描述符布局 |
| 性能分析 | `tracy` | GPU/CPU性能分析 |

### Phase 1: 基础框架 (1-2周)

- [ ] Buffer/Image RAII封装 + VMA集成
- [ ] 基础Shader加载 + 三角形渲染

### Phase 2: 资源管理 (1-2周)

- [ ] 简单模型加载(Obj/GLTF)

### Phase 3: RenderGraph (2周)
- [ ] 资源声明系统
- [ ] 自动屏障/布局转换
- [ ] Transient Resource内存复用
- [ ] 可视化调试输出

### Phase 4: 渲染技术 (渐进式学习路线)

Phase 4 采用**递进式学习**策略，每种 GI 方法都是后续技术的基础。所有实现都基于 RenderGraph，支持实时切换对比。

#### 4.0 基础渲染框架
- [x] Forward Rendering基础（已完成）
- [ ] Deferred Rendering（G-Buffer + Lighting Pass）
- [ ] PBR材质系统（Metallic-Roughness工作流）
- [ ] Shadow Mapping（PCSS/CSM级联阴影）
- [ ] Post-processing基础（Bloom/Tonemap/Gamma校正）

#### 4.1 屏幕空间技术（Screen-Space Techniques）
**目的**：学习屏幕空间光线追踪，这是 HSGI 的核心基础

- [ ] **SSAO** (Screen-Space Ambient Occlusion)
  - GTAO 实现（比 SSAO 更准确）
  - Horizon-Based Ambient Occlusion
  - Multi-Bounce AO 近似

- [ ] **SSR** (Screen-Space Reflections)
  - Ray Marching in Screen Space
  - Binary Search 优化
  - Roughness 处理（Cone Tracing 近似）

- [ ] **Hi-Z Buffer** (Hierarchical Z-Buffer) ⭐ **HSGI关键依赖**
  - Depth Pyramid 构建（Compute Shader）
  - Hi-Z Tracing（对数步长光线追踪）
  - 层次遍历优化（从 coarse 到 fine）

- [ ] **SSS** (Screen-Space Shadows / Contact Shadows)
  - 短距离高精度阴影
  - 与 CSM 混合策略

#### 4.2 Probe-based GI（基于探针的全局光照）
**目的**：理解现代实时 GI 的核心思想，为 HSGI 的稀疏采样打基础

- [ ] **Irradiance Volumes** (经典方法)
  - 规则网格放置 Probe
  - SH (Spherical Harmonics) 编码光照
  - 三线性插值采样

- [ ] **DDGI** (Dynamic Diffuse Global Illumination) ⭐ **当前主流方案**
  - 自适应 Probe 放置（基于表面密度）
  - Probe 滚动更新（Rolling Update）
  - Radiance 存储与插值
  - Probe 可见性处理（Backface Culling）
  - **学习价值**：理解 Probe GI 的完整 pipeline

- [ ] **Reflection Probes**
  - Parallax Correction
  - Box/Projection 投影
  - Blend Weight 计算

#### 4.3 体素化 GI（Voxel-based Techniques）
**目的**：理解世界空间稀疏结构，HSGI 的 "Sparse" 部分

- [ ] **SVOGI** (Sparse Voxel Octree GI)
  - 保守光栅化体素化（Conservative Rasterization）
  - Sparse Voxel Octree 构建
  - Voxel Cone Tracing（Diffuse + Specular）

- [ ] **LPV** (Light Propagation Volumes) - 已计划
  - 直接光注入
  - 迭代传播（Propagation Iterations）
  - 阻塞处理（Blocking Potentials）

#### 4.4 采样与重要性采样（Sampling）
**目的**：高质量 GI 需要高质量采样

- [ ] **Blue Noise Sampling**
  - Blue Noise 纹理生成/加载
  - 时序抖动（Temporal Jittering）
  - 低差异序列（Hammersley、Halton）

- [ ] **RIS** (Resampled Importance Sampling)
  - Reservoir 数据结构
  - WIS (Weighted Importance Sampling)
  - **预备知识**：为 ReSTIR 打基础

- [ ] **ReSTIR DI** (Direct Illumination) - 可选进阶
  - 多光源场景的重要性采样
  - Spatiotemporal Reservoir Reuse
  - 去相关策略（Visibility Reuse）

- [ ] **ReSTIR GI** (Indirect Illumination) - 可选进阶
  - 路径复用（Path Reuse）
  - Temporal Path 追踪
  - 与 Probe 结合的可能性

#### 4.5 HSGI 实现（最终目标）
**HSGI = Hierarchical + Sparse + GI**，整合前面所有技术

前置要求（必须完成）：
- ✅ Hi-Z Buffer（层次追踪）
- ✅ Probe-based GI 经验（稀疏采样）
- ✅ Screen-Space Ray Tracing（SSR）
- ✅ DDGI 或 Irradiance Volumes（光照编码）

实现步骤：

- [ ] **HSGI - Phase 1: Hierarchical Screen-Space Ray Tracing**
  - 结合 Hi-Z + SSR 的远距离反射
  - Roughness 自适应步长

- [ ] **HSGI - Phase 2: Sparse Probe Placement**
  - 基于相机视角的 Probe 分布
  - LOD 层级（近处密集，远处稀疏）
  - Hierarchical Probe 查询

- [ ] **HSGI - Phase 3: Unified GI Framework**
  - 近距离：Screen-Space Tracing (Hi-Z)
  - 中距离：Sparse Probe Tracing
  - 远距离：Voxel/Cube Map Fallback
  - 统一的重要性采样框架

- [ ] **HSGI - Phase 4: Advanced Features**
  - Specular GI（Glossy Reflections）
  - Multi-Bounce Diffuse（多次反弹近似）
  - 时序稳定性（Temporal Filtering）
  - 动态物体支持（Moving Objects）

## GPU Profiler：性能分析与优化工具

GPU Profiler是渲染开发的**必备工具**，用于测量GPU时间线、定位性能瓶颈。对于实现HSGI等复杂GI技术，Profiler帮助你理解每毫秒的代价。

### 为什么需要GPU Profiler？

**场景**：HSGI跑了30fps，想知道瓶颈在哪里
- ❌ 猜测："可能是采样太多？"
- ✅ Profiler："Hi-Z构建占3ms，Probe更新占8ms，光线追踪占15ms → 优化追踪"

### GPU Profiler核心功能

| 功能 | 说明 | 用途 |
|------|------|------|
| **GPU Time Markers** | 在命令流中插入时间戳 | 测量每个Pass的耗时 |
| **Pipeline Statistics** | 统计三角面数、draw call数 | 检查是否过度绘制 |
| **Memory Bandwidth** | 追踪纹理/缓冲区读取 | 优化数据布局 |
| **Occupancy** | Shader核心利用率 | 优化寄存器占用 |
| **Frame Timeline** | 可视化CPU/GPU并行 | 发现同步等待 |

### 常用GPU Profiler工具

#### 1. **Tracy**（推荐，免费开源）
```cpp
// C++侧标记
ZoneScopedN("GBuffer Pass");

// Vulkan侧标记（使用VK_EXT_debug_utils）
vkCmdWriteTimestamp(cmd, VK_PIPELINE_STAGE_BOTTOM_OF_PIPE_BIT, queryPool, queryIndex);

// 结果在Tracy GUI中实时显示
```
**优势**：
- CPU + GPU 统一时间线
- 集成简单（头文件+库）
- 支持远程分析（PC→Android）
- 低开销（适合实时调试）

#### 2. **RenderDoc**（免费，帧捕获）
- 捕获单帧的所有命令
- 查看每个Draw Call的纹理/缓冲区内容
- 检查Render Target结果
- **Texture Inspector**：查看SSR/Hi-Z中间结果

#### 3. **NVIDIA Nsight Graphics**（N卡专用）
- GPU Speed of Light：理论峰值vs实际
- Shader Profiler：逐指令分析
- Ray Tracing Profiler：RT Core利用率

#### 4. **Radeon GPU Profiler**（A卡专用）
- 硬件队列可视化
- Barrier/Wait分析
- 适用于AMD优化

### Tracy在Vulkan中的使用示例

```cpp
// 初始化
void Context::initProfiler() {
    // Tracy需要VK_EXT_calibrated_timestamps扩展
    m_profilerContext = TracyVkContextCalibrated(
        physicalDevice, device, graphicsQueue, cmdBuffer,
        vkGetPhysicalDeviceCalibrateableTimeDomainsEXT,
        vkGetCalibratedTimestampsEXT
    );
}

// 每帧标记
void DeferredRenderer::render() {
    TracyVkZoneC(m_profilerContext, cmdBuffer, "DeferredFrame", TRACY_COLOR_GREEN);
    
    // GBuffer Pass
    {
        TracyVkZoneC(m_profilerContext, cmdBuffer, "GBuffer", TRACY_COLOR_BLUE);
        vkCmdBeginRenderPass(cmd, ...);
        // ... 绘制几何体
        vkCmdEndRenderPass(cmd);
    }
    
    // Lighting Pass
    {
        TracyVkZoneC(m_profilerContext, cmdBuffer, "Lighting", TRACY_COLOR_YELLOW);
        vkCmdBeginRenderPass(cmd, ...);
        // ... 计算光照
        vkCmdEndRenderPass(cmd);
    }
    
    // HSGI Pass
    {
        TracyVkZoneC(m_profilerContext, cmdBuffer, "HSGI", TRACY_COLOR_RED);
        
        TracyVkZoneC(m_profilerContext, cmdBuffer, "HiZ Build", TRACY_COLOR_PINK);
        m_hiZ->buildPyramid(cmd);
        
        TracyVkZoneC(m_profilerContext, cmdBuffer, "Probe Trace", TRACY_COLOR_CYAN);
        m_hsgi->traceProbes(cmd);
    }
}

// 在Tracy GUI中看到的时间线：
// |--GBuffer(2.1ms)--|--Lighting(0.8ms)--|--HSGI(8.5ms)--|--Present--|
//                          └─HiZ Build(1.2ms) └─Probe Trace(6.8ms) ← 瓶颈在这里！
```

```cpp

    builder.execute([](CommandBuffer& cmd, auto& res) {

```

```cpp
    TracyVkZone(cmd, ("HiZ_Level_" + std::to_string(level)).c_str());
    vkCmdDispatch(cmd, ...);
}
```

#### 3. **采样统计**（算法调优）
```cpp
// 统计每条光线的步进次数（用于优化Hi-Z）
buffer uint rayStepCounts[];
// Compute Shader中：atomicAdd(rayStepCounts[rayId], 1);

```

### 性能优化的数据驱动方法


### 特别章节：CUDA到Vulkan RT的迁移（双轨开发策略）

由于用户已有CUDA Path Tracer经验，建议**并行双轨开发**：

**轨道1：软件光追（学习原理）**
- Hi-Z SSR、DDGI Probe捕获 - 用软件实现深入理解算法

**轨道2：硬件光追（参考验证）**
- 将CUDA Path Tracer快速迁移到Vulkan RT
- 作为**Ground Truth**验证软件光追结果
- 支持实时切换对比（Split Screen模式）

#### Vulkan Ray Query vs CUDA 对照

| CUDA | Vulkan | 说明 |
|------|--------|------|
| `optixTrace` | `rayQueryProceedEXT` | Compute Shader内联追踪 |
| `__closesthit__` | `closesthit` shader | Ray Pipeline模式 |
| `__miss__` | `miss` shader | 未命中处理 |
| `launchParams` | Push Constants / UBO | 参数传递 |
| `CUdeviceptr` | BufferDeviceAddress | GPU指针访问 |
| `curand` | Blue Noise / PCG | 随机数生成 |

#### 推荐：Ray Query模式（Compute Shader）

```glsl
// 与CUDA最接近的模式，无需复杂Shader Binding Table
#version 460
#extension GL_EXT_ray_query : require

layout(local_size_x = 8, local_size_y = 8) in;
layout(set = 0, binding = 0) uniform accelerationStructureEXT tlas;
layout(set = 0, binding = 1, rgba8) uniform writeonly image2D outputImage;

void main() {
    ivec2 pixel = ivec2(gl_GlobalInvocationID.xy);
    vec2 uv = (vec2(pixel) + 0.5) / imageSize(outputImage);
    
    // 生成光线（与CUDA相同逻辑）
    vec3 origin = ...;
    vec3 direction = ...;
    
    // Ray Query - 就像CUDA的optixTrace
    rayQueryEXT rq;
    rayQueryInitializeEXT(rq, tlas, gl_RayFlagsOpaqueEXT, 0xFF,
                          origin, 0.01, direction, 1000.0);
    
    while(rayQueryProceedEXT(rq)) {
        // 遍历BVH，可处理任意命中
    }
    
    // 获取结果
    if (rayQueryGetIntersectionTypeEXT(rq, true) == gl_RayQueryCommittedIntersectionTriangleEXT) {
        vec2 bary = rayQueryGetIntersectionBarycentricsEXT(rq, true);
        // ... 着色计算
    }
}
```

1. **GLSL化你的采样函数**（几乎直接翻译）

## 技术实现关键点

### GI 技术的关键依赖关系

```
HSGI (最终目标)
├── Hi-Z Buffer (层次追踪基础)
│   └── SSR (屏幕空间反射)
├── Probe-based GI (DDGI)
│   ├── Irradiance Volumes (基础概念)
│   └── SH Encoding (球谐编码)
├── Sparse Voxel (稀疏体素)
│   └── LPV / SVOGI
└── Sampling (高质量采样)
    ├── Blue Noise
    └── RIS/ReSTIR (可选)
```

### 每种技术的核心学习目标

| 技术 | 核心概念 | 为 HSGI 提供的价值 |
|------|----------|-------------------|
| **SSR** | Screen-Space Ray Marching | HSGI 近距离追踪基础 |
| **Hi-Z** | Hierarchical Traversal | HSGI "Hierarchical" 核心 |
| **DDGI** | Probe Placement & Interpolation | HSGI "Sparse" 采样策略 |
| **SVOGI** | Voxelization & Cone Tracing | 世界空间稀疏结构 |
| **ReSTIR** | Importance Sampling | 高质量样本生成（可选） |

### RenderGraph 资源命名规范

为了支持多种 GI 技术的对比和混合，建议统一的资源命名：

```cpp
// G-Buffer 资源（所有技术共享）
"GBuffer.Albedo"
"GBuffer.Normal"      // World-space normal
"GBuffer.Depth"       // Linear depth
"GBuffer.Material"    // Roughness/Metallic/AO
"GBuffer.Velocity"    // 用于TAA

// 中间结果
"SSAO.AO"
"SSR.Reflection"
"DDGI.Irradiance"
"HiZ.Pyramid"         // Mip 0 = full-res depth

// 最终输出
"Lighting.Direct"
"Lighting.Indirect"
"PostProcess.Final"
```

### 技术切换与混合策略

```cpp
// 使用 Technique Layer 支持运行时切换
enum class GITechnique {
    None,

```
