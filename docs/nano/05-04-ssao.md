# 05-04：SSAO 后处理 Pass

**What**: 新增屏幕空间环境光遮蔽（Screen-Space Ambient Occlusion）Pass，并接入延迟渲染管线；同时修复了 `PrefilterEnvPass` 描述符池大小 warning。

**Why**:
- 之前的 PBR 光照只有直接光 + IBL 环境光，缺少接触阴影和缝隙遮挡，模型看起来有点“飘”。
- SSAO 是延迟管线的经典后处理，复用 GBuffer 的 Normal/Depth 即可计算，实现成本低。

**How**:

1. **新增 `SSAOPass`**（`src/pass/SSAOPass.h/.cpp` + `shaders/ssao.comp`）
   - 输入 `GBuffer Normal`（binding 1）和 `GBuffer Depth`（binding 2）。
   - 输出 `SSAO` 纹理（1280×720，`R8_UNORM`，storage + sampled）。
   - Compute shader 在 view space 重建像素位置，用法线构造切空间半球，随机旋转后采样 16 个点。
   - 每个采样点投影回屏幕读取 depth，再重建 view-space 位置；只有在 sample point **前方** 且在半径范围内的表面才被计入遮挡。
   - 背景（`depth >= 1.0`）直接输出 `ao = 1.0`。

2. **新增 `SSAOBlurPass`**（`src/pass/SSAOBlurPass.h/.cpp` + `shaders/ssao_blur.comp`）
   - 对原始 SSAO 做 5×5 Gaussian blur，平滑噪声并弱化 silhouette 硬边。
   - 输出 `SSAOBlurred`（`R8_UNORM`）。

3. **管线集成**（`src/technique/DeferredShading.h/.cpp`）
   - 管线顺序：`GBuffer -> ShadowMap -> SSAO -> SSAOBlur -> Lighting -> LightVisualize -> Present`。
   - `LightingPass` 读取 blurred AO，并通过 UI 开关 `Enable SSAO` 控制。

4. **LightingPass 接入 SSAO**（`src/pass/LightingPass.h/.cpp` + `shaders/lighting.frag`）
   - descriptor binding 9 绑定 `ssaoSampler`。
   - 新增 `ssaoEnabled` push constant。
   - SSAO 只乘到 diffuse ambient 项，`specular IBL` 不受影响，避免金属/反射出现奇怪的泛光。
   - SSAO 未启用时绑定 1×1 白色占位 AO，保证 descriptor set 始终合法。

5. **修复 `PrefilterEnvPass` 描述符池 warning**（`src/precompute/PrefilterEnvPass.cpp`）
   - 原 pool 只分配 1 个 `COMBINED_IMAGE_SAMPLER`，但实际要分配 `m_mipLevels` 套 set。
   - 改为 `descriptorCount = m_mipLevels`，消除 validation warning。

6. **目录修正**
   - `SSAOPass` 不是预计算 Pass，从 `src/precompute/` 移回 `src/pass/`。
   - `BRDFLutPass` 继续保留在 `src/precompute/`。

**Trade-offs**:
- ✅ 基于 GBuffer，不需要额外几何 pass，与延迟管线天然契合。
- ✅ view-space 距离判断 + 修正后的遮挡符号，避免了仰视时地面被错误大面积压暗。
- ❌ 当前 SSAO 没有 bilateral blur，5×5 Gaussian blur 会轻微模糊接触阴影边缘；后续可改双边滤波保留几何边界。
- ❌ 参数（radius / bias / sample count）目前硬编码在 shader 中，后续可暴露到 UI 或 push constant。
- ❌ Silhouette 处仍有淡淡的亮边，是 screen-space 采样本身的固有 artifact，增加采样数和 blur 半径可进一步减弱。

**验证**:
1. `cmake --build build --config Debug` 通过，新 shader 被编译。
2. 运行 Dragon 场景（`assets/scenes/dragon-ssao.json`）：
   - 开启 SSAO 后，龙身缝隙、与地面接触处有柔和阴影。
   - 仰视/平视时地面不再被错误压暗，环境 cubemap 背景无变化。
   - 金属/反射区域没有明显泛光。
3. 无新增 VVL 错误。
